## 実装済みの大きな方針

### 1. AI provider

- 採点 provider は `auto / mock / gemini / ollama`
- 本番では `gemini-2.5-flash` 前提
- Anthropic は不要判断で削除済み

### 2. prompt 方針

- 採点 prompt とコメント prompt の分離は、今回は見送り
- 理由:
  - 70 人規模で API コールが実質 2 倍になる
  - Gemini free tier を考えると運用が重い
- 現在は 1 回の API 呼び出しで
  - 5 項目スコア
  - ポジティブ 3 文
  - 改善 1 文
  を返す方式

### 3. caption

- UI には caption 入力なし
- 採点経路にも caption は渡していない
- ただし DB / repository 上の古い field は残してある
- 本番前は安全性優先でそのまま

### 4. 参加者向けフィードバック

- 結果発表前は `/join/{token}` で見えない
- `presentation` 画面から公開 / 非公開を切り替える
- 公開後に `/join/{token}` を開くと、最後に投稿した写真の結果が見える

現在の設計:

- 採点は全員に対して行う
- ranking / presentation は対象者だけ表示
- 3 位までが片側だけにならないよう、ルールベースの自動点数調整を入れる
- ランキング対象者は `最終確定点` をそのまま見せる
- ランキング対象外の人は `min(最終確定点, 3位未満の上限)` を見せる

コメント構成:

- 通常のゲスト表示:
  - ポジティブ 1 文
  - 改善 1 文
- 上位 3 人:
  - 1 位: ポジティブ 3 文
  - 2 位: ポジティブ 2 文 + 改善 1 文
  - 3 位: ポジティブ 1 文 + 改善 1 文

### 5. 管理画面

- 投稿一覧は
  - `ランキング対象の投稿`
  - `ランキング対象外の投稿`
  に分離済み
- `表示順位` 機能は不要判断で削除済み
- 順位を動かしたい場合は `点数補正` を使う前提

### 6. フォント

- 名前表示まわりは `person-name` を使って調整済み
- PC では修正確認済み
- ただしスマホは未確認
- 本質的な課題は:
  - `髙𫞎` のような名前で
  - カタカナ / 通常漢字 / 異字体 `𫞎` が別フォント fallback になりやすい
  - 名前全体を 1 つの統一フォントで見せたい
